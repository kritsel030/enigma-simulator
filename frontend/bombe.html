Turing-Welchman bombe

We received this encoded message: C E C A A E

We expect the original message to contain C A F E

C E C A A E
C A F E
C A F E ← this one works!
C A F E

1 2 3 4
C E C A A E
C A F E ← this one works!
This results in the menu A - 2 - C - 1 - E - 4 - A - 3 - F

You should read this as:

  • an enigma in start position + 2 rotor steps will encode A into C and vice versa
  • an enigma in start position + 1 rotor step will encode C into E and vice versa
  • an enigma in start position + 4 rotor steps will encode E into A and vice versa
  • an enigma in start position + 3 rotor steps will encode A into F and vice versa
If we would have guessed the correct rotor settings and plugboard connections, we would have these three Enigmas.
(notice that the bottom rotor setting is different for each of the Enigma's).
